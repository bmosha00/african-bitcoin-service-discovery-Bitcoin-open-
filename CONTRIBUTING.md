# Contributing

This protocol is open infrastructure for African Bitcoin payment companies. Contributions are welcome from alliance members and the broader community.

## How to contribute

### Spec changes
1. Open an issue describing the change
2. Fork the repo and create a branch
3. Submit a pull request with your proposed changes
4. Alliance members review and discuss
5. Changes merged after rough consensus

### New tag values
New `rail_out`, `network`, or other tag values can be proposed via GitHub issue. If no objections after 7 days, they're added to the data model reference.

### Reference implementations
The publisher and querier libraries live in the lib/ directory. Bug fixes and improvements welcome via pull request.

## Discussion

- **Alliance Signal group** — Primary discussion channel for active members
- **GitHub Issues** — For spec questions, proposals, and bug reports
- **GitHub Discussions** — For open-ended questions and ideas

## Code of conduct

Be respectful. We're building infrastructure for a continent. Disagreements about technical decisions are welcome. Personal attacks are not.
